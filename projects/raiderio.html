Raider IO

Raider IO

Raider IO is a PVP online game up to 20 players, where each one can choose between three different classes to fight their friends on an online arena. Each class has it's own unique Spell, creating a different type of gameplay with each class.

Killing an enemy will increase the player level, making them bigger and stronger, but slower! If the player gets killed, they will lose their level and start from scratch, also, the killer will steal the level and grow.

This game was made with my colleague Marc Guillen, for the subject "Networks and Online Games" of the Bachelor's Degree in Video Game Design and Development at the UPC.

On this project we had to develop our own Network System for the game engine in C++ using UDP and implement different kinds of systems to handle input latency, packet loss and others.

My Contribution

Common Features

The majority of the network part was developed by both of us doing pair programming. The list of features we developed together are the following:

  • Server/Client connection
  • Data Serialization
  • Connection Timeout: If the Server or the Client doesn't receive a Ping Packet in a set amount of time, a Timeout will be considered and a Disconnect will be executed.
  • World State Replication with Managers: The Server sends replication packets to the Clients every set amount of time and the Clients read those packets succesfully and replicate the state of the server.
  • Reliability for UDP (Delivery Manager and Input Packet Redundancy): Input Packet Redundancy makes sure the server receives all the input packets in order. Delivery Manager recognizes when a packet was or not succesfull and calls a delegate to handle.
  • Client Side Prediction with Server Reconciliation for Movement: The input of the Client is handled client side making it feel smooth and react quick. When a Replication Packet is received by the Client, the state is recovered through Server Reconciliation reapplying all the inputs accordingly.
  • Entity Interpolation

My Features

The rest of features that I implemented by myself are:

  • Player Structure
  • Player Movement
  • Spawn Weapon and Link on Client
  • Die/Respawn
  • Class Selection
  • Animation Serialization
  • Client Side Prediction for Projectiles (Fake Projectile on Client): This creates a "Fake" projectile on the client for smoother inputs, this projectile will follow the trajectory it think the server will follow also, creating an ilusion for the client. The fake projectile will get destroyed on contact with another Player but will not do any damage, as it is the Server Simulation of the Real Projectile who will deal with the damage event.
  • Hunter Spell
  • Handle Destruction of Child Network Objects of Client
  • Level System
  • Helped with Mouse Input Serialization
  • Death Animation / Sound
  • Class Helmets Art
